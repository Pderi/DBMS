## 8. 多表查询

### 测试目标
验证多表连接查询和嵌套查询的实现。

### 测试用例

#### 测试8.1: 内连接（INNER JOIN）
```sql
SELECT s.name, c.course_name, e.grade
FROM students s
INNER JOIN enrollments e ON s.id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id;
```
**预期结果**: 返回学生姓名、课程名和成绩的关联数据

#### 测试8.2: 左连接（LEFT JOIN）
```sql
SELECT s.name, e.grade
FROM students s
LEFT JOIN enrollments e ON s.id = e.student_id;
```
**预期结果**: 返回所有学生，即使没有选课记录也显示（grade为NULL）

#### 测试8.3: 右连接（RIGHT JOIN）
```sql
SELECT c.course_name, e.grade
FROM enrollments e
RIGHT JOIN courses c ON e.course_id = c.course_id;
```
**预期结果**: 返回所有课程，即使没有选课记录也显示（grade为NULL）

#### 测试8.4: 嵌套查询 - IN子查询
```sql
SELECT * FROM students 
WHERE id IN (SELECT student_id FROM enrollments WHERE grade > 90);
```
**预期结果**: 返回成绩大于90分的学生信息

#### 测试8.5: 嵌套查询 - NOT IN子查询
```sql
SELECT * FROM students 
WHERE id NOT IN (SELECT student_id FROM enrollments);
```
**预期结果**: 返回没有选课的学生信息

#### 测试8.6: 嵌套查询 - 子查询作为条件
```sql
SELECT * FROM courses 
WHERE course_id IN (
    SELECT course_id FROM enrollments 
    WHERE grade > 85
);
```
**预期结果**: 返回有学生成绩大于85分的课程

#### 测试8.7: 嵌套查询 - 子查询作为列
```sql
SELECT s.name, 
       (SELECT COUNT(*) FROM enrollments e WHERE e.student_id = s.id) AS course_count
FROM students s;
```
**预期结果**: 返回每个学生及其选课数量

## 11. 事务机制

### 测试目标
验证事务机制（BEGIN TRANSACTION、COMMIT、ROLLBACK）的实现。

### 测试用例

#### 测试11.1: 事务提交（COMMIT）
```sql
BEGIN TRANSACTION;
INSERT INTO students VALUES (10, 'TestUser', 25, 'Male', 'test@example.com');
UPDATE students SET age = 26 WHERE id = 10;
COMMIT;
```
**预期结果**: 
- 事务开始
- 插入和更新操作执行
- 事务提交，所有更改永久保存

#### 测试11.2: 事务回滚（ROLLBACK）
```sql
BEGIN TRANSACTION;
INSERT INTO students VALUES (11, 'RollbackTest', 30, 'Female', 'rollback@example.com');
SELECT * FROM students WHERE id = 11;
ROLLBACK;
SELECT * FROM students WHERE id = 11;
```
**预期结果**: 
- 事务开始
- 插入操作执行
- 第一次查询能看到新记录
- 回滚后，第二次查询找不到该记录（已回滚）

#### 测试11.3: 事务中的多个操作
```sql
BEGIN TRANSACTION;
INSERT INTO courses VALUES (10, 'Test Course', 2, 'Test Teacher');
UPDATE courses SET credits = 3 WHERE course_id = 10;
DELETE FROM courses WHERE course_id = 10;
COMMIT;
```
**预期结果**: 所有操作在事务中执行，提交后最终状态正确

#### 测试11.4: 嵌套事务（如果支持）
```sql
BEGIN TRANSACTION;
INSERT INTO students VALUES (12, 'Nested', 20, 'Male', 'nested@example.com');
BEGIN TRANSACTION;
UPDATE students SET age = 21 WHERE id = 12;
COMMIT;
ROLLBACK;
```
**预期结果**: 根据系统实现，验证嵌套事务的处理方式

#### 测试11.5: 事务中的查询
```sql
BEGIN TRANSACTION;
SELECT * FROM students;
INSERT INTO students VALUES (13, 'QueryTest', 22, 'Female', 'query@example.com');
SELECT * FROM students;
ROLLBACK;
SELECT * FROM students;
```
**预期结果**: 
- 事务中能看到未提交的更改
- 回滚后，更改被撤销
